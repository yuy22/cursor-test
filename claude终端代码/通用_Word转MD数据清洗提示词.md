# 通用 Word → Markdown 数据清洗提示词

## 使用方式

把下面这段话发给 Claude，替换 `【】` 内的占位符即可。

---

## 提示词正文

```
我有以下 Word 文档需要转换为供 AI 分析的 Markdown 文件：

【粘贴文件路径，每行一个】

Anthropic API Key：【你的 Key】
输出目录：【输出根目录路径】

请按以下两步处理，使用现有工具和脚本，不要重新发明轮子：

## 准备工作：先检查现有脚本

在 Desktop\claude终端代码\ 下查找是否已有可复用的脚本（如 docx_to_md.py、clean_docx.py 等），
有则在其基础上修改，没有再新建。

---

## 第一步：字号分析（必须先做）

对每个文件运行字号分析，输出各字号的出现次数和示例文本，
据此确定每个文件的标题层级映射（SIZE_TO_HEADING），
不要猜测，要用实际数据说话。

---

## 第二步：docx → Markdown 转换

### 核心规范（不可违反）

1. 用 python-docx 读取文档
2. 遍历方式：必须按文档原始顺序遍历 doc.element.body，同时处理段落(p)和表格(tbl)
3. 标题识别：优先用字号分析得出的 SIZE_TO_HEADING 映射，其次用样式名(Heading 1/2/3)
4. 图片提取：用 zipfile 直接解压 docx 的 word/media/ 目录，保存到输出目录的 images/ 子目录
5. 图片引用：必须用 get_para_image_names() 方式获取（见下方代码），禁止用占位符

get_para_image_names() 实现：
- 方式1：遍历 a:blip 元素，取 r:embed 属性 → 查关系表 → 得到真实文件名
- 方式2：遍历 v:imagedata 元素，取 r:id 属性 → 查关系表 → 得到真实文件名
- 注意：必须用完整命名空间 URI，不能用 qn('v:imagedata')（会报 KeyError）

6. 图片在 MD 中暂时输出为：![图N](images/真实文件名)，第三步再替换

### 如果是 OCR 扫描版文档，额外进行文本清洗

判断依据：文本中出现大量乱码、字符间多余空格、水印/页码/出版信息。

清洗规则（参考 Desktop\clean_docx.py 的实现）：
- 丢弃：水印行、纯数字页码行、全大写拼音行、出版社信息行（ISBN/CIP/责任编辑/定价等）
- 丢弃：孤立人名行（纯汉字+内部空格，总长 ≤15 字）、书名页眉页脚重复行
- 修复：去掉行尾页码（|001 格式）、压缩 OCR 汉字间多余空格、压缩连续空格

---

## 第三步：图片 → 文字描述（Claude Vision API）

### 处理原则

- 所有图片都必须过 Vision API，一张都不跳过
- 模型：claude-haiku-4-5-20251001（省钱）
- 格式无法识别的图片（WMF/EMF）：保留原 ![图N](...) 引用，不报错退出

### 给 Vision API 的提示词

请分析这张图片，按以下规则回复：

1. 如果是【装饰图】（纯边框、背景色块、空白分隔线、无实际内容）：
   只回复：DECORATIVE

2. 如果是【数学公式或数学图形】（含算式、运算符、几何图、坐标轴、分数、根号等）：
   回复格式：
   MATH
   （用 LaTeX 表达公式，行内用 $...$，块级用 $$...$$；无法用 LaTeX 表达则用中文描述数学含义）

3. 如果是【教学插图、示意图、图表、照片、表格截图】：
   回复格式：
   DIAGRAM
   这是...（一句中文描述，不超过 80 字）

### 替换规则

| Vision 结果 | MD 中的处理 |
|---|---|
| DECORATIVE | 删除整行，不留任何引用 |
| MATH | 替换为 LaTeX：$公式$ 或 $$公式$$ |
| DIAGRAM | 替换为：> [图片内容] 这是... |
| WMF/EMF（不支持） | 保留原 ![图N](...) 不动 |

---

## 输出要求

- 每个文档输出到独立子目录：输出根目录/文件名/
- 子目录结构：文件名.md + images/（存放原始图片）
- 脚本保存到：Desktop\claude终端代码\（命名能看出处理的文档）
- 运行完打印统计：处理图片总数、MATH 数量、DIAGRAM 数量、DECORATIVE 删除数量

---

## 验证清单（完成后逐项确认）

- [ ] MD 文件无乱码
- [ ] 标题层级与原文结构一致
- [ ] 所有 ![图N](...) 已被替换，无残留
- [ ] 数学公式以 LaTeX 格式呈现，可正常渲染
- [ ] 装饰图已删除，无空白引用行
- [ ] OCR 噪声行已清除（如适用）
- [ ] images/ 目录图片数量与 docx 内嵌图片数量一致
```
