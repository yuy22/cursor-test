## 三、清洗规则说明

### 3.1 噪声过滤（自动删除）

| 规则 | 示例 |
|---|---|
| 水印行（多种变体） | "仅供个人科研教学使用！" |
| 纯页码行 | "001" "002" |
| 全大写拼音行 | "ZHONGZI KE YIGE..." |
| 出版/版权关键词行 | "责任编辑：xxx" |
| ISBN / CIP 数据行 | "ISBN 978-7-..." |
| OCR 合并的出版团队行 | "出版人策划编辑项目统筹..." |
| 系列丛书行 | "教育家书院丛书·研究系列" |
| 孤立人名行（有内部空格的短行） | "刘 灿" / "何 薇 郑 莉" |
| 免责声明行 | "如有印装质量问题..." |
| 出版社信息表格（整表跳过） | 含"出版发行"/"社址"的表格 |
| 广告水印 | "更多教学好书请加微信：xxx" |
| 页眉重复行（章节标题单独成行） | 正文中重复出现的章标题 |

> **二次检测**：`fix_text()` 压缩空格后再过一遍噪声规则，
> 防止"出 版 人 策划编辑"这类被空格撑开的行漏网。
> 第一遍跑完后必须抽查，发现漏网噪声用正则二次清洗。

### 3.2 文本修复 + 格式标准化

| 问题 | 修复方式 |
|---|---|
| 章节末尾页码 "写在前面\|001" | 正则去掉 `\|\d+` 后缀 |
| OCR 汉字间多余空格 "种 子 课" | 检测中文占比>50% 且有单字间隔，则压缩空格 |
| 多余连续空格 | 压缩为单个空格 |
| 半角标点（中文语境） | `,` → `，`、`?` → `？`、`!` → `！`、`()` → `（）` |

### 3.3 结构识别（转为 Markdown 标题）

| 样式 | Markdown 级别 | 示例 |
|---|---|---|
| `一、二、三...` 开头的短行 | `##` | "二、种子课：用生长替代重复" |
| `*` 或 `·` 开头的导语行（含≥2中文字） | `###` | "* 导 语" |
| `（一）（二）` 括号编号行 | `####` | "（一）教学目标分析" |
| `——` 开头的副标题 | `>` 引用块 | "——以《用字母表示数》为例" |
| `—` 开头的引文署名（短行） | `>` 引用块 | "—叶圣陶" |

### 3.4 表格处理

- 出版社信息表格：**自动跳过**（第一格含"出版发行/社址"等关键词）
- 内容表格：转为标准 Markdown 表格格式
- 空白表格（有效单元格 < 3）：自动跳过

### 3.5 去重

- 段落级去重：`seen_lines` 集合检测重复段落（>10字符）
- 连续重复行检测：相邻两行相同则只保留一行
- 页眉重复行：章标题在正文中反复出现时，作为噪声删除

## 四、图片处理策略（v2 新增）

### 4.1 核心原则

- **不是全删也不是全留** — 按类型分类处理
- **文字描述 + 保留原图路径** — 文字保证 RAG 能搜到，原图留着以后用
- **忠实原文** — 看不清就写"扫描模糊"，绝不编造

### 4.2 自动分类规则

| 过滤条件 | 判定 | 处理 |
|---|---|---|
| 面积 < 5000px² | 装饰碎片（1px线、点） | 删除 |
| 长宽比 > 8:1 | 分隔线、边框线 | 删除 |
| 平均亮度 > 230 | 水印残片、空白扫描页 | 删除 |
| 平均亮度 < 10 且面积 < 50000px² | 纯黑装饰块 | 删除 |
| 以上都不满足 | 可能有内容 | 保留 |

### 4.3 人工审查（脚本跑完后必做）

自动分类会有漏网之鱼，跑完后必须：

1. 用 Read 逐张查看保留的图片
2. 分三类：
   - **垃圾**（书脊、空白页、灰色装饰块）→ 删除整行
   - **装饰可转文字**（签名、章节标题装饰）→ 替换为文字
   - **有教学价值**（插图、图表、公式图）→ 写描述 + 保留原图
3. 用 Bash 脚本批量替换 MD 中的图片引用

### 4.4 图片描述格式

```markdown
[图片描述：网格纸上的梯形，左半部分用斜线阴影填充，演示通过分割计算梯形面积的方法]
![图片](images/rId109.jpeg)
```

### 4.5 PDF 交叉参考

扫描模糊的图片，可以导出 PDF 对应页来辅助确认内容：

```python
import fitz  # pymupdf
doc = fitz.open("file.pdf")
page = doc[页码]
pix = page.get_pixmap(dpi=150)
pix.save("page_XXX.png")
```

**但必须忠实原文，不能产生幻觉。**

## 五、数学符号处理策略

| 符号类型 | 处理方式 | 原因 |
|---|---|---|
| `√ × ÷ ± → ℃ ≤ ≥ ≠` 等 Unicode | **直接保留** | Markdown/所有 RAG 系统都支持 |
| OMML 公式（Word 内嵌公式对象） | v2 可提取为图片 + 描述 | python-docx 不原生解析 |
| LaTeX 分数 `$\frac{1}{10}$` | 转中文语义"十分之一（1/10）" | 中文语义更利于 RAG 检索 |
| LaTeX 字符串 | 直接保留 `$...$` 格式 | 已是文本，无需转换 |

**如遇到 OMML 公式（`<m:oMath>` 标签）的处理思路：**

```python
# 检测是否含有 OMML
full_xml = doc.element.xml
has_omml = '<m:oMath' in full_xml
# 如果有，需要用 lxml + XSLT 转换为 LaTeX，或者安装 mammoth 库
```

## 六、RAG 注意事项

- **不去停用词** — 现代 Embedding 模型需要完整自然语言，去停用词反而降低检索质量
- **图片在标准 RAG 框架中不会被传给模型** — 所以必须在清洗阶段就写好文字描述
- **保留原图路径** — 未来 RAG 框架支持多模态时可直接用

## 九、常见问题

**Q：中文路径报错 PackageNotFoundError？**
A：不要用 bash heredoc 传路径，改用 `Write` 工具把 Python 脚本写成文件再运行。

**Q：某些人名/短行没被过滤掉（如"曲凤玲"）？**
A：二次清洗阶段用 `^[\u4e00-\u9fff]{2,3}$` 匹配孤立2-3字汉字行。但要注意保护标题行（以 # 开头的不删）。

**Q：表格内容有重复列？**
A：DOCX 合并单元格在 python-docx 中会重复，属正常现象，不影响语义。

**Q：图片过滤后还有垃圾图？**
A：自动分类不可能100%准确，必须人工逐张审查保留的图片，手动删除漏网垃圾。

**Q：扫描质量太差看不清图片内容？**
A：用 pymupdf 导出 PDF 对应页查看，PDF 通常比 Word 内嵌图片更清晰。但描述必须忠实，看不清就写"扫描模糊"。
