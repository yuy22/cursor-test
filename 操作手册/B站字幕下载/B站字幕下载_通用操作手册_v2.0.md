## 🎯 AI应该记住的规则

### 【时长规则】
- **最小时长**：≥23分钟（1380秒）
- **原因**：10分钟左右的是课堂片段，完整课程需要UP主私信
- **执行**：检查到<23分钟自动跳过，显示理由

### 【重复检查】
- **规则**：同标题只下载一次
- **判定方式**：完全相同的标题视为重复
- **执行**：用集合记录已下载标题，发现重复自动跳过

### 【字幕检查】
- **要求**：视频必须有AI生成的中文字幕
- **来源**：B站API `/x/player/wbi/v2?bvid=XX&cid=XX`
- **执行**：无字幕自动跳过

### 【成功条件】
- **目标**：成功下载3个视频
- **执行**：达到3个就停止，不继续扫描
- **输出**：统计[成功个数][被跳过个数][总扫描数]

### 【标点恢复规则】优先级

| 1 | 已有标点 | 保留 |

| 4 | 末尾感叹词+间隔≥0.5秒 | ！ |
| 5 | 间隔≥0.8秒 | 。 |
| 6 | 其他 | ， |

## 🔧 B站API配置（AI内部使用）

### 核心API端点
| 用途 | 端点 | 需Cookie | 说明 |
|------|------|:---:|------|
| 搜索视频 | `/x/web-interface/search/type` | 否 | 获取前10播放量最高 |
| 获取CID | `/x/player/pagelist?bvid={BV}` | 否 | 分P视频需要 |
| 获取字幕 | `/x/player/v2?bvid={BV}&cid={CID}` | **是** | 必须SESSDATA+bili_jct+buvid3 |
| 获取时长 | `/x/web-interface/view?bvid={BV}` | 否 | duration字段 |
| 登录验证 | `/x/web-interface/nav` | **是** | 检查isLogin字段 |

### ⚠️ API关键注意事项（2026-03-15验证）

**字幕API选择**：
- ✅ **用 `/x/player/v2`**（无wbi前缀）— 能正确返回字幕
- ❌ **不用 `/x/player/wbi/v2`** — 即使WBI签名正确也返回空（原因不明）

**必须验证登录态**：
- 每次运行前先调 `/x/web-interface/nav` 检查 `isLogin` 是否为 `true`
- `isLogin=false` → SESSDATA已过期，必须让用户重新提供
- 过期SESSDATA不会报错，只是静默返回空字幕，极易误判为"视频无字幕"

**必须验证字幕内容**：
- `/v2` 端点在SESSDATA过期或异常时会返回**错误视频的字幕**（张冠李戴）
- 下载后必须抽查前几句内容是否与视频主题匹配
- 如果内容明显不对（数学课字幕出现电竞/美食等），说明登录态有问题

## ⚠️ 已知限制与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| **SESSDATA过期（最常见！）** | 有效期约30天 | **让用户重新提供SESSDATA+bili_jct+buvid3** |
| SESSDATA过期的症状 | API不报错，但返回空字幕或错误字幕 | 先调`/nav`检查`isLogin`再干活 |
| 老视频无AI字幕 | 2022年前的视频B站未生成AI字幕 | 无解，需用户的Chrome字幕插件手动导出 |
| `/v2`返回错误字幕 | 登录态异常时返回其他视频的字幕 | 抽查内容是否匹配，不匹配就是Cookie问题 |
| 某些视频搜索失败 | B站限流（412错误） | 改用WebFetch或改关键词 |
| 视频无AI字幕 | B站未自动生成 | 自动跳过，继续扫描 |
| SSL连接重置 | 请求频繁被B站CDN断开 | 等30秒重试，已下载的不会丢 |
| 标点准确率70-80% | 纯规则方案局限 | 可接受，或后续接NLP模型 |

### 🔴 AI每次运行前必做的检查清单
1. 读取SESSDATA缓存文件
2. 调 `/x/web-interface/nav` 验证 `isLogin=true`
3. 如果 `isLogin=false` → **立即告诉用户：SESSDATA过期了，请提供新的SESSDATA、bili_jct、buvid3**
4. 获取方法提示用户：Chrome → bilibili.com → Ctrl+Shift+I → Application → Cookies → .bilibili.com
5. 不要在登录失败的情况下尝试下载（会拿到错误数据）

## 📌 2026-03-15 踩坑记录（重要经验）

### 踩坑1: SESSDATA过期不报错
- **症状**：API返回正常（code=0），但字幕列表为空
- **误判**：以为视频没有AI字幕
- **真相**：SESSDATA过期了，`/nav`接口`isLogin=false`
- **教训**：每次运行前必须先验证登录态

### 踩坑2: `/v2` 端点返回错误字幕
- **症状**：下载了24个文件，打开发现内容是电竞解说、iPhone评测
- **原因**：`/x/player/v2` 在认证异常时返回随机视频的字幕（张冠李戴）
- **教训**：下载后必须抽查内容，不能盲信API返回

### 踩坑3: `/wbi/v2` 需要WBI签名但仍返回空
- **症状**：实现了完整WBI签名（img_key+sub_key+mixin+w_rid），API仍返回0字幕
- **结论**：对于某些视频，`/wbi/v2` 就是不工作；**用 `/v2`（无wbi）代替**

### 踩坑4: 2020年老视频没有AI字幕
- **案例**：BV1e7411o7qT（2020年CETV"同上一堂课"51P）
- **原因**：B站AI字幕约2022-2023年上线，老视频未回溯处理
- **视频里的文字**：是电视台烧录在画面中的硬字幕，非可提取文本
- **解决**：这类老视频只能靠Chrome字幕插件（OCR方式）提取

### 踩坑5: 请求频繁导致SSL连接重置
- **症状**：`SSL: UNEXPECTED_EOF_WHILE_READING` 或 `ConnectionResetError 10054`
- **原因**：短时间内请求过多，B站CDN强制断开
- **解决**：等30秒再重试，每个请求间隔1秒，已下载文件支持断点续跑
