## 步骤4：标点恢复（核心算法）— [更新于2026-03-13]

### 做了什么
B站AI字幕是纯语音转录，没有任何标点。利用字幕条目间的时间间隔（说话人的自然停顿）和末尾词语境规则，自动推断标点。

### 标点判定规则（按优先级）

| 优先级 | 条件 | 标点 | 说明 |
|:---:|------|:---:|------|
| 1 | 已有标点 `。！？，、；：…—` | 保留 | 不重复添加 |
| 2 | 间隔 < 0.3秒 | ， | 同一句内，连续语流 |
| 3 | 末尾匹配疑问词 | ？ | 见疑问词表 |
| 4 | 末尾是感叹词 + 间隔≥0.5秒 | ！ | "好""棒""厉害"等 |
| 5 | 间隔 ≥ 0.8秒 | 。 | 长停顿，句子结束 |
| 6 | 其他（0.3-0.8秒） | ， | 短停顿，逗号 |

### 疑问词尾匹配列表
```
吗|呢|吧|啊|呀|么|嘛|没|不|谁|什么|怎么|怎样|哪|几|多少|
为什么|为啥|是不是|对不对|好不好|能不能|行不行|可不可以|
有没有|对吗|是吧|懂了没|明白了吗|听懂了吗|听明白了|看懂了吗
```

### 感叹词尾列表
```
好|对|棒|厉害|漂亮|真棒
```

### 已知局限
- 纯规则方案，准确率约70-80%，个别位置会有误判
- "好"既可以是感叹也可以是"好的"过渡，容易误标
- 连续快语速段（间隔<0.1秒）全部标逗号，可能少断句
- 若需更高精度，可考虑接入NLP标点恢复模型（FunASR的ct-punc等），但需额外依赖

## 步骤1：获取SESSDATA

### 做了什么
B站AI字幕API需要登录态Cookie，核心是 `SESSDATA` 字段。脚本按优先级获取：环境变量 → 缓存文件 → 用户手动输入。首次获取后自动缓存到 `.bili_sessdata` 文件，后续无需重复操作。

### SESSDATA获取方法（需告知用户）
1. Chrome打开 bilibili.com（确保已登录）
2. **Ctrl+Shift+I** 打开开发者工具（如果F12无效，也可右键→检查）
3. 顶部标签栏点 **Application**（可能被折叠在 `>>` 里）
4. 左侧栏 **Cookies** → 点击 `.bilibili.com`
5. 找到 **SESSDATA** 行，双击Value列复制

### 缓存位置
```
C:\Users\b886855456ly\Desktop\claude终端代码\.bili_sessdata
```

### 注意事项
- SESSDATA有效期约30天，过期后需重新获取
- 如果API返回"无字幕数据"，首先检查SESSDATA是否过期
- Windows下 `browser_cookie3` 需要管理员权限（VSS影子复制），非管理员环境走手动输入
- SESSDATA值包含URL编码字符（如 `%2C`），复制时保持原样不要解码

## 步骤2：解析视频信息（BVID → CID）

### 做了什么
B站每个视频有两个ID：`BVID`（视频级）和 `CID`（分P级）。字幕API同时需要两者。BVID可从URL直接提取，CID需要调API获取。

### 从URL提取BVID
```
https://www.bilibili.com/video/BV1t4PGemEhw/  →  BVID = BV1t4PGemEhw
https://www.bilibili.com/video/BV1xxxxx/?p=3   →  BVID = BV1xxxxx, 第3P
```
正则提取：`BV[a-zA-Z0-9]+`

### 从BVID获取CID（API）
```
GET https://api.bilibili.com/x/player/pagelist?bvid={BVID}
```
- 无需Cookie
- 返回JSON，`data` 是数组，每项包含 `cid`（数字）、`part`（分P标题）、`page`（第几P）
- 单P视频：`data[0]["cid"]`
- 多P视频：按 `page` 字段匹配

### 从合集/Season获取全部视频（批量场景）
```
GET https://api.bilibili.com/x/polymer/web-space/seasons_archives_list
    ?mid={UP主MID}&season_id={合集ID}&page_num=1&page_size=100
```
- 返回 `data.archives` 数组，每项包含 `bvid`、`title`、`cid`（注意：此处cid即为第一P的cid）
- `page.total` 表示总数，超过100需翻页

### 核心参数获取方法

| 参数 | 获取方式 |
|------|----------|
| BVID | 视频URL中直接提取 `BV...` |
| CID | 调 `pagelist` API，或合集API的 `archives` 字段 |
| MID（UP主ID） | UP主主页URL：`space.bilibili.com/{MID}` |
| Season ID | 合集页面URL参数，或在UP主主页"合集"标签下的链接中提取 |

## 步骤3：下载字幕JSON

### 做了什么
调B站player API获取字幕列表，拿到字幕文件URL后下载JSON。

### API端点
```
GET https://api.bilibili.com/x/player/wbi/v2?bvid={BVID}&cid={CID}
```

### 必须条件
- **Headers**：需要 `User-Agent`（模拟Chrome）和 `Referer`（B站视频页）
- **Cookies**：必须携带 `SESSDATA`，否则返回空字幕列表

### 响应结构（关键路径）
```json
{
  "code": 0,
  "data": {
    "subtitle": {
      "subtitles": [
        {
          "lan": "ai-zh",
          "subtitle_url": "//aisubtitle.hdslb.com/bfs/ai_subtitle/prod/...",
          "lan_doc": "中文（自动生成）"
        }
      ]
    }
  }
}
```

### 字幕选择规则
1. 优先选 `lan` 包含 `"zh"` 的（中文字幕）
2. 若无中文，取第一条
3. `subtitle_url` 以 `//` 开头，需补 `https:`

### 字幕JSON结构
```json
{
  "body": [
    {
      "from": 7.66,      // 开始时间（秒）
      "to": 8.86,        // 结束时间（秒）
      "sid": 1,           // 序号
      "content": "上课起立"  // 文本内容
    }
  ]
}
```

### 关键特征
- AI字幕**没有任何标点符号**，是纯语音转文字
- `from`/`to` 时间戳是标点恢复和分段的核心依据
- 每条字幕通常1-15个字，对应1-3秒语音

## 步骤4：标点恢复（核心算法）

### 做了什么
B站AI字幕是纯语音转录，没有标点。利用字幕条目间的时间间隔（说话人的自然停顿）和末尾词语境规则，自动推断标点。

### 标点判定规则

| 条件 | 标点 | 说明 |
|------|:---:|------|
| 间隔 < 0.3秒 | ， | 同一句内，连续语流 |
| 末尾匹配疑问词 | ？ | 见疑问词表 |
| 末尾是感叹词 + 间隔≥0.5秒 | ！ | "好""棒""厉害""真棒"等 |
| 间隔 ≥ 0.8秒 | 。 | 长停顿，句子结束 |
| 其他（0.3-0.8秒） | ， | 短停顿，逗号 |
| 已有标点 | 保留 | 不重复添加 |

### 疑问词尾匹配列表

### 已知局限

## 步骤5：输出MD文件
